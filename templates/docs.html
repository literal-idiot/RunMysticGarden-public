Base URL

http://localhost:5000

All API requests should be made to this base URL.

Authentication

Register User
POST

Endpoint: /auth/register

Description: Create a new user account

Request Body:
{
    "email": "user@example.com",
    "username": "runner123",
    "password": "securepassword123"
}
Response:
{
    "message": "User registered successfully",
    "access_token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
    "user": {
        "id": 1,
        "email": "user@example.com",
        "username": "runner123",
        "created_at": "2025-06-24T10:00:00Z",
        "is_active": true
    }
}
Login
POST

Endpoint: /auth/login

Description: Authenticate user and get access token

Request Body:

Runs

Log Run
POST

Endpoint: /api/runs

Description: Log a new running session

Authentication: Required

Request Body:
{
    "distance_km": 5.2,
    "duration_minutes": 30,
    "intensity": "moderate"
}

Intensity levels: low, moderate, high, extreme

Response:
{
    "message": "Run logged successfully",
    "run": {
        "id": 1,
        "distance_km": 5.2,
        "duration_minutes": 30,
        "intensity": "moderate",
        "pace_min_per_km": 5.77,
        "coins_earned": 62,
        "created_at": "2025-06-24T10:30:00Z"
    },
    "coins_earned": 62,
    "total_coins": 162
}
Get Runs
GET

Endpoint: /api/runs

Description: Get user's running history

Authentication: Required

Query Parameters:
  • page - Page number (default: 1)
  • per_page - Items per page (default: 20, max: 100)

Statistics

Get User Statistics
GET

Endpoint: /api/stats

Description: Get comprehensive user statistics

Authentication: Required

Response includes:
  • User profile information
  • Running statistics (total distance, runs, averages)
  • Wallet information
  • Garden level and plant statistics

Error Handling

The API uses conventional HTTP response codes to indicate success or failure:

  • 200 OK - Request succeeded
  • 201 Created - Resource created successfully
  • 400 Bad Request - Invalid request data
  • 401 Unauthorized - Authentication required
  • 404 Not Found - Resource not found
  • 409 Conflict - Resource already exists
  • 500 Internal Server Error - Server error
Error Response Format:
{
    "error": "Detailed error message describing what went wrong"
}
